# custom.financia — Design Spec

**Date:** 2026-05-26
**Status:** Approved
**Approved by:** Raul Belloso Medina

---

## 1. Problem Statement

The business needs a tool for data-entry operators ("capturistas") to record bank account transactions — deposits, withdrawals, and the running balance — as fast as possible. The existing `atlas.finance` double-entry accounting module is too complex for this use case. The goal is something that feels like a bank register in Excel: fast, tabular, keyboard-driven.

---

## 2. Solution Summary

A new AME3 custom module (`custom.financia`) that provides:
- Multiple bank accounts (MXN or USD, one currency per account)
- A spreadsheet-mode register where all rows are inline-editable, auto-saving on row blur
- Auto-computed running balance (server-side window function)
- PDF, Excel, and CSV export
- CSV/XLSX import with column-mapping wizard
- Per-account summary page with balance-over-time and category-breakdown charts
- Category and transaction-type management

## 4. Architecture

**Hybrid AME3:** Use `defineModel` for all data models (Atlas ORM manages DB tables). Use standard `TABLE`/`FORM` blueprint views for simple management screens (accounts list, categories, types). Use `CUSTOM` views for the three complex UI pieces — SpreadsheetRegister, AccountSummary, ImportWizard — which are custom React components registered in `modules/custom/custom.financia/components/`.

The API follows the service factory closure pattern established by `custom.fleet`. The export service reuses the PDFKit + ExcelJS patterns from `apps/api/src/services/ledger-export-service.js`.

## 5. Data Models

### `financia.account` → `financia_account`
```
id              UUID v7 (auto)
company_id      UUID
name            text(255)      e.g. "BBVA Principal"
bank            text(255)      e.g. "BBVA"
account_number  text(64)       last 4 or full
currency        select         ["MXN","USD"]
opening_balance decimal(18,4)
enabled         boolean
created_at / updated_at
```

### `financia.transaction_type` → `financia_transaction_type`
```
id, company_id
code  text(32)    e.g. "SPEI","CHQ","EFE","SAT","TRASPASO"
name  text(128)
enabled
```

### `financia.category` → `financia_category`
```
id, company_id
name  text(128)
color color(32)   hex, for charts
kind  select      ["income","expense","both"]
enabled
```

### `financia.transaction` → `financia_transaction`
```
id              UUID v7
account_id      UUID → financia_account
company_id      UUID
fecha           date
tipo_id         UUID nullable → financia_transaction_type
numero          text(64) nullable
nombre          text(255)
referencia      text(255) nullable
concepto        text(512) nullable
deposito        decimal(18,4) nullable
retiro          decimal(18,4) nullable
category_id     UUID nullable → financia_category
enabled         boolean
created_at / updated_at
```

**Computed (not stored):**
- `consecutive` = `ROW_NUMBER() OVER (PARTITION BY account_id ORDER BY fecha, created_at)`
- `saldo_actual` = `account.opening_balance + SUM(COALESCE(deposito,0)-COALESCE(retiro,0)) OVER (PARTITION BY account_id ORDER BY fecha, created_at ROWS UNBOUNDED PRECEDING TO CURRENT ROW)`

Both computed server-side. Editing a past-dated transaction automatically corrects all subsequent balances on next fetch.

## 7. Screens

### Accounts list (`/accounts`)
Standard `AtlasCrudView` — TABLE + FORM blueprints. Columns: Name, Bank, Account #, Currency, Current balance (from summary), Status.

### Account detail (`/accounts/:id`)
CUSTOM `AccountScreen` with two tabs:
- **Registro:** Period filter bar + toolbar (`[Importar]` `[Exportar PDF]` `[Exportar Excel]` `[Exportar CSV]`) + SpreadsheetRegister
- **Resumen:** KPI cards + line chart (balance over time) + bar/donut chart (by category)

### Import wizard (`/accounts/:id/import`)
CUSTOM `ImportWizard` — 3 steps: upload → column mapping → preview + commit

### Categories and Types
Standard `AtlasCrudView` (TABLE + FORM) for both.

## 8. SpreadsheetRegister

Custom React component using a plain `<table>` with controlled `<input>`/`<select>` cells. No third-party spreadsheet library.

**Columns:** `[#] [Fecha] [Tipo] [Numero] [Nombre] [Referencia] [Concepto] [Deposito] [Retiro] [Saldo] [⋯]`
- `#` and `Saldo` are read-only computed columns
- `Tipo` and `Categoria` are inline `<select>` dropdowns

**Auto-save:** Row saves on blur (focus leaves the entire row). POST for new rows, PATCH for existing.

**Keyboard shortcuts:**
- Tab / Shift+Tab — next/prev editable cell
- Enter — save row, move to next row same column
- Escape — cancel cell edit
- ↑ / ↓ — move between rows
- F2 — enter edit mode
- Ctrl+Enter — force save without leaving
- Ctrl+Z — undo last saved change

## 10. Export

Reuses patterns from `apps/api/src/services/ledger-export-service.js`:
- PDF: PDFKit, A4, header + table + footer totals
- Excel: ExcelJS, two sheets (Movimientos + Resumen), frozen headers, currency formatting
- CSV: UTF-8 BOM, same columns as Excel sheet 1

---

## 11. Import

Formats: CSV and XLSX.
- XLSX parsing: ExcelJS `workbook.xlsx.read()`
- CSV parsing: `csv-parse` (add dependency)
- Column mapping UI: user matches detected headers to model fields
- Preview: server validates each row, returns per-row error flags
- Commit: single DB transaction, all valid rows

---

## 12. Charts (Resumen tab)

Library: Recharts (add dependency to `apps/desktop`).
- Line chart: `balance_series` — daily closing balance
- Bar/donut chart: `by_category` — deposits and withdrawals per category

## 15. Critical AME3 Rules

1. Never edit `prisma/schema.prisma` for `financia_*` tables
2. Never use `prisma.<model>` accessors — always `prisma.$queryRaw` tagged templates
3. All service functions inside `createXxxService({ prisma })` factory closure
4. Never generate UUIDs in JavaScript — use `INSERT ... RETURNING *`
