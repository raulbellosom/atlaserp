# Phase 9 - HR Module Plan

Date: 2026-05-05
Spec: `docs/superpowers/specs/2026-05-05-phase9-hr-design.md`

## Objective

Ship HR v1 as a dedicated-route employee experience with complete visible profile sections, markdown notes, file dossier, and embedded audit timeline.

## Workstreams

1. Data model expansion
- Extend `HrEmployee` with approved v1 fields (`employeeCode`, employment/contact details, dates, status).
- Add forward Prisma migration only (no edits to existing migrations).

2. Validators and contracts
- Extend/create HR validators for create/update/enabled flows.
- Add field-level constraints for email/date/status formats.

3. API service layer
- Expand `hr-service` to support full v1 payload shape.
- Preserve company scope and `enabled` lifecycle.
- Emit audit events for create/update/enable/disable/note/file actions.

4. API routes
- Keep HR endpoints thin and service-driven:
  - `GET /hr/employees`
  - `GET /hr/employees/:id`
  - `POST /hr/employees`
  - `PUT /hr/employees/:id`
  - `PATCH /hr/employees/:id/enabled`
  - `GET /hr/employees/:id/audit`

5. SDK integration
- Normalize `atlas.hr` methods for list/get/create/update/enabled/audit.
- Keep request/response shape consistent with existing module domains.

6. Module and routing integration
- Ensure `atlas.hr` manifest permissions/navigation/blueprints are registered.
- Route list and dedicated employee detail in `ModuleOutlet`.

7. Desktop UI (dedicated record page)
- Build list -> detail navigation flow for employees.
- Employee detail as single long-form page with visible blocks:
  - identity/actions
  - employment
  - contact
  - documents
  - notes
  - history
- Add View/Edit toggle, save/cancel, and stable layout behavior.

8. Notes and markdown rendering
- Keep editor in edit mode and rich rendering in read mode.
- Avoid raw markdown syntax display in read mode.

9. Files integration
- Upload multiple files from employee page using `/files/upload`.
- Persist HR linkage metadata (`moduleKey/entityType/entityId`).
- Show dossier list with download/preview actions.

10. Observability and UX feedback
- Add explicit loading/disabled states for slow operations.
- Add success/failure toasts in Spanish.

## Verification

1. Syntax and build checks
- `node --check apps/api/src/index.js`
- `node --check apps/api/src/services/hr-service.js`
- `node --check packages/validators/src/index.js`
- `node --check packages/sdk/src/index.js`
- `pnpm.cmd --filter ./apps/desktop build:web`

2. Manual smoke
- Install/enable HR module from catalog.
- Create employee -> open dedicated route.
- Toggle view/edit -> save changes -> verify timeline entry.
- Upload multiple files -> verify list and download action.
- Confirm notes render formatted markdown in read mode.

## Completion criteria

1. Employee dedicated route fully functional and stable.
2. Full HR v1 field set persists correctly through API/SDK/UI.
3. Files and audit are visible in the same employee page.
4. `docs/TASKS.md` updated with validated Phase 9 checklist and evidence.
